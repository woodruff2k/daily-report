# 영업 일보 시스템 요건 정의서

## 1. 시스템 개요

영업 담당자가 매일의 활동을 보고하고, 상급자가 피드백을 하기 위한 영업 일보 관리 시스템

## 2. 기능 요건

### 2.1 일보 작성 기능

- 영업 담당자는 하루 1건의 일보를 작성할 수 있다
- 일보에는 다음을 기재:
  - 당일의 과제·상담 사항(Problem)
  - 익일의 활동 계획(Plan)
  - 작성 일시

### 2.2 방문 기록 등록 기능

- 하나의 일보에 대해 복수의 고객 방문 기록을 등록 가능
- 각 방문 기록에는 다음을 기재:
  - 방문한 고객
  - 방문 내용
  - 방문 시각(임의)

### 2.3 상급자 코멘트 기능

- 상급자는 일보에 대해 코멘트를 추가할 수 있다
- 코멘트는 여러 번 추가 가능
- 코멘트 일시를 기록

### 2.4 마스터 관리

- 영업 마스터: 영업 담당자의 정보를 관리
- 고객 마스터: 고객 정보를 관리

## 4. 테이블 정의 상세

### 4.1 영업 마스터(SALES_PERSON)

- **주키**: sales_person_id
- **용도**: 영업 담당자 및 관리자의 정보를 관리
- **특기 사항**: is_manager 플래그로 관리자 권한을 식별

### 4.2 고객 마스터(CUSTOMER)

- **주키**: customer_id
- **용도**: 방문 대상이 되는 고객 정보를 관리
- **특기 사항**: 회사 정보와 담당자 정보를 일원 관리

### 4.3 일보(DAILY_REPORT)

- **주키**: report_id
- **용도**: 하루 1건의 일보 본체를 관리
- **특기 사항**:
  - sales_person_id와 report_date의 조합으로 유니크 제약을 설정
  - problem과 plan은 필수 항목

### 4.4 방문 기록(VISIT_RECORD)

- **주키**: visit_id
- **용도**: 일보에 연결된 개별 고객 방문 기록
- **특기 사항**:
  - 하나의 일보에 복수의 방문 기록을 등록 가능
  - 방문 시각은 임의 항목

### 4.5 상급자 코멘트(MANAGER_COMMENT)

- **주키**: comment_id
- **용도**: 관리자에 의한 일보 피드백
- **특기 사항**:
  - 하나의 일보에 복수의 코멘트 가능
  - manager_id는 SALES_PERSON 테이블의 관리자를 참조

## 5. 추가 검토 사항

### 5.1 권한 관리

- 영업 담당자: 자신의 일보만 편집 가능
- 관리자: 휘하의 영업 담당자의 일보를 열람·코멘트 가능

### 5.2 유효성 검증(Validation)

- 동일 일자의 일보는 1인 1건까지
- 과거 일자의 일보는 작성으로부터 일정 기간 후에는 편집 불가

### 5.3 향후 확장성

- 방문 기록에 성과(수주액 등)의 추가
- 일보의 승인 워크플로
- 월차·분기 리포트의 자동 생성

## 6. 비기능 요건(향후 상세화)

### 6.1 성능 요건

- 일보 목록의 표시는 3초 이내
- 동시 접속 사용자 수: 100명 상정

### 6.2 보안 요건

- 로그인 인증 필수
- 통신은 HTTPS화
- 액세스 로그의 기록

### 6.3 운용 요건

- 백업: 일차
- 가동 시간: 평일 8:00-20:00(메인터넌스 시간 제외)
