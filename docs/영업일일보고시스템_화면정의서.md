## 2\. 화면별 상세 정의

### SCR-100 로그인

- **설명**: 사번/이메일과 비밀번호로 시스템에 로그인한다.  
- **접근 권한**: 전체(비로그인)

**레이아웃**

```
+------------------------------------+
|        영업 일일 보고 시스템        |
|                                    |
|   [ 이메일/사번        ]           |
|   [ 비밀번호           ]           |
|   [ ] 로그인 상태 유지             |
|        [    로그인    ]            |
+------------------------------------+
```

**화면 항목**

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 이메일/사번 | Text | Y | 영업 마스터에 등록된 값 |
| 비밀번호 | Password | Y |  |
| 로그인 상태 유지 | Checkbox | N |  |

**버튼/액션**: \[로그인\] — 인증 성공 시 SCR-200 이동

**유효성·처리**: 인증 실패 시 오류 메시지 표시, 5회 실패 시 잠금 안내. 인증/세션은 서버에서 검증.

---

### SCR-200 일일보고 목록 (메인)

- **설명**: 로그인 사용자의 일일보고를 기간/상태별로 조회한다. 오늘자 보고가 없으면 작성 버튼을 노출한다.  
- **접근 권한**: 영업사원/상급자(본인 보고)

**레이아웃**

```
+--------------------------------------------------------+
| [기간 ____~____] [상태 ▼] [검색]      [+ 오늘 보고 작성] |
+--------------------------------------------------------+
| 보고일자  | 방문건수 | 상태   | 댓글 | 최종수정       |
| 06-20    |   3     | 제출   |  2  | 18:10  [상세]  |
| 06-19    |   2     | 작성중 |  0  | 17:40  [상세]  |
+--------------------------------------------------------+
```

**화면 항목 (목록)**

| 항목 | 설명 |
| :---- | :---- |
| 보고일자 | report\_date |
| 방문건수 | 해당 보고의 방문기록 수 |
| 상태 | 작성중(DRAFT) / 제출(SUBMITTED) |
| 댓글 | 댓글 건수 |
| 최종수정 | updated\_at |

**검색 조건**: 기간(시작\~종료), 상태

**버튼/액션**

- \[+ 오늘 보고 작성\] → 오늘자 보고 신규 생성 후 SCR-210 (이미 있으면 해당 보고로 이동)  
- \[상세\] → SCR-220

---

### SCR-210 일일보고 작성·수정

- **설명**: 방문기록(고객+내용)을 여러 행으로 입력하고, 과제/상담(Problem)과 내일 할 일(Plan)을 작성한다.  
- **접근 권한**: 영업사원(본인, DRAFT 상태)

**레이아웃**

```
+---------------------------------------------------------+
| 보고일자: 2026-06-20      작성자: 홍길동      상태: 작성중 |
+---------------------------------------------------------+
| [ 방문 기록 ]                              [+ 행 추가]    |
|  시각 | 고객(검색) | 방문유형 | 방문내용 | 상담결과 | [삭제]|
| 10:00 | (주)A상사  | 방문    | ...      | 견적요청 | [x]   |
| 14:00 | B마트     | 전화    | ...      | 보류     | [x]   |
+---------------------------------------------------------+
| [ 과제/상담(Problem) ]                     [+ 행 추가]   |
|  관련고객 | 내용 | 상태 | [삭제]                         |
+---------------------------------------------------------+
| [ 내일 할 일(Plan) ]                        [+ 행 추가]  |
|  관련고객 | 예정일 | 내용 | [삭제]                       |
+---------------------------------------------------------+
|              [임시저장]   [제출]                          |
+---------------------------------------------------------+
```

**① 방문기록 (VISIT\_RECORD) — 다중 행**

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 방문시각 | Time | N |  |
| 고객 | Search/Select | Y | 고객 마스터 검색 선택 |
| 방문유형 | Select | Y | 방문/전화/온라인 |
| 방문내용 | Textarea | Y |  |
| 상담결과 | Text | N |  |

**② 과제/상담 (REPORT\_PROBLEM) — 다중 행**

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 관련고객 | Search/Select | N | 선택 |
| 내용 | Textarea | Y |  |
| 상태 | Select | N | OPEN/CLOSED (기본 OPEN) |

**③ 내일 할 일 (REPORT\_PLAN) — 다중 행**

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 관련고객 | Search/Select | N | 선택 |
| 예정일 | Date | N | 기본값: 익일 |
| 내용 | Textarea | Y |  |

**버튼/액션**

- \[+ 행 추가\] / \[삭제\] — 각 섹션 행 동적 추가·삭제  
- \[임시저장\] — DRAFT 상태 저장 (재편집 가능)  
- \[제출\] — 유효성 통과 시 SUBMITTED 전환, SCR-220 이동

**유효성 검증**

- 방문기록 최소 1행 이상, 각 행 고객·방문유형·방문내용 필수  
- 제출 후에는 수정 정책에 따라 잠금(편집 불가) 또는 재상신 처리

### SCR-220 일일보고 상세·조회

- **설명**: 제출된 보고를 조회한다. 상급자는 이 화면에서 댓글로 의견을 남긴다.  
- **접근 권한**: 영업사원(본인) / 상급자(팀원)

**레이아웃**

```
+---------------------------------------------------------+
| 보고일자 2026-06-20  작성자 홍길동  상태 제출  제출 18:10 |
+---------------------------------------------------------+
| [방문 기록]   (읽기 전용 테이블)                          |
| [과제/상담]   (읽기 전용)                                 |
| [내일 할 일]  (읽기 전용)                                 |
+---------------------------------------------------------+
| [ 댓글 / 상급자 의견 ]                                    |
|  └ 김부장: 견적 일정 확인 바람   (06-20 19:00)  [답글]    |
|     └ 홍길동: 확인했습니다       (06-20 19:10)            |
|  [ 댓글 입력 ........................ ] [등록]            |
+---------------------------------------------------------+
```

**화면 항목 (조회 영역)**: SCR-210의 방문/과제/계획을 읽기 전용으로 표시

**댓글 (REPORT\_COMMENT)**

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 댓글 내용 | Textarea | Y | 상급자 작성 (대댓글로 작성자 응답 가능) |
| 답글 | Action | \- | 특정 댓글에 대댓글 작성 |

**버튼/액션**: \[등록\] 댓글 저장 / \[답글\] 대댓글 작성 / \[수정·삭제\] 본인 댓글 한정

**처리**: 댓글 작성 권한은 서버에서 검증(상급자 또는 본인). 작성 시 작성자·시각 자동 기록.

---

### SCR-300 팀 보고 조회 (상급자)

- **설명**: 상급자가 소속 팀원의 보고를 사원별/기간별/고객별로 검색·조회한다.  
- **접근 권한**: 상급자

**검색 조건**

| 항목 | 유형 | 설명 |
| :---- | :---- | :---- |
| 팀원 | Select | 소속 팀원 다중 선택 |
| 기간 | Date Range | 보고일자 기준 |
| 고객 | Search | 방문 고객 기준 |
| 상태 | Select | 제출/작성중 |

**버튼/액션**: 목록 행 \[상세\] → SCR-220

---

### SCR-400 고객 마스터 목록

- **설명**: 고객 마스터를 검색·조회하고 등록 화면으로 진입한다.  
- **접근 권한**: 영업사원/상급자

**검색 조건**: 고객명/회사명, 담당 영업, 등급, 상태(ACTIVE/INACTIVE)

**목록 항목**: 고객명, 회사명, 연락처, 등급, 담당 영업, 상태

**버튼/액션**: \[+ 신규 등록\] → SCR-410 / 행 \[수정\] → SCR-410

---

- **접근 권한**: 영업사원/상급자

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 고객/담당자명 | Text | Y |  |
| 회사명 | Text | N |  |
| 연락처 | Text | N | 형식 검증 |
| 이메일 | Text | N | 형식 검증 |
| 주소 | Text | N |  |
| 고객등급 | Select | N |  |
| 담당 영업 | Select | Y | 영업 마스터 참조 |
| 상태 | Select | Y | ACTIVE/INACTIVE |

**버튼/액션**: \[저장\] / \[취소\] / \[비활성화\]

### SCR-500 영업 마스터 목록

- **설명**: 영업사원 계정을 검색·조회한다.  
- **접근 권한**: 관리자

**검색 조건**: 이름/사번, 부서, 상태

**목록 항목**: 사번, 이름, 부서, 직급, 상급자, 상태

- **접근 권한**: 관리자

| 항목 | 유형 | 필수 | 설명·제약 |
| :---- | :---- | :---: | :---- |
| 사번 | Text | Y | 유일값 |
| 이름 | Text | Y |  |
| 이메일 | Text | Y | 로그인 ID, 유일값 |
| 부서 | Text/Select | N |  |
| 직급 | Text/Select | N |  |
| 상급자 | Select | N | 영업 마스터(자기참조) |
| 상태 | Select | Y | ACTIVE/INACTIVE |

**버튼/액션**: \[저장\] / \[취소\] / \[비활성화\]

## 3\. 공통 정책

- **상태 전이**: 일일보고 DRAFT → SUBMITTED. 제출 후 수정 정책은 운영 기준에 따름.  
- **마스터 비활성화**: 물리 삭제 대신 상태값으로 비활성화하여 과거 보고의 참조 무결성을 유지.  
- **권한 검증**: 모든 조회·작성·댓글 권한은 서버 측에서 검증(화면 비표시는 접근통제 아님).  
- **이력 관리**: 생성·수정 일시 및 작성자 자동 기록.
